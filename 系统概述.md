# 📋 闲置物品慧选平台 - 系统概述

## 1. 系统简介
本系统是一个基于 Spring Boot 3 + Vue 3 的校园/社区闲置物品交易平台。旨在建立一个可信的二手交易环境，支持完整的商品发布、浏览、搜索、下单、支付模拟、发货收货及评价流程。

---

## 2. 核心业务功能

### （1）用户与账号模块
| 功能          | 描述                   | 逻辑要点                    |
| :------------ | :--------------------- | :-------------------------- |
| **注册/登录** | 账号密码注册、JWT 登录 | 密码加密存储、Token自动刷新 |
| **个人信息**  | 修改昵称、头像、邮箱   | 头像上传至对象存储 (OSS)    |
| **收货地址**  | 多地址管理             | 支持增删改查，设置默认地址  |
| **安全中心**  | 修改密码、注销账号     | 敏感操作需二次验证          |

### （2）商品模块 (卖家视角)
| 功能         | 描述                             | 逻辑要点                                 |
| :----------- | :------------------------------- | :--------------------------------------- |
| **发布商品** | 填写标题、描述、价格、分类、成色 | 支持多图上传，初始状态为"待审核"或"上架" |
| **商品管理** | 编辑信息、手动上下架、删除       | 仅"待审核/上架"状态可编辑                |
| **状态流转** | 待审核 → 上架 → 交易中 → 已售    | 交易中锁定库存，不可被其他人购买         |

### （3）商品模块 (买家视角)
| 功能         | 描述                               | 逻辑要点            |
| :----------- | :--------------------------------- | :------------------ |
| **首页推荐** | 最新发布商品瀑布流                 | 分页加载            |
| **分类浏览** | 树形分类检索 (如 数码/手机/iPhone) | 支持多级分类        |
| **高级搜索** | 关键词 + 价格范围 + 成色筛选       | 支持按价格/时间排序 |
| **商品详情** | 展示大图、卖家信息、留言互动       | 记录浏览量          |

### （4）交易模块 (核心闭环)
| 功能          | 描述                        | 逻辑要点                                          |
| :------------ | :-------------------------- | :------------------------------------------------ |
| **购物车**    | 加入购物车、修改数量、结算  | 校验商品状态是否有效                              |
| **提交订单**  | **选择收货地址** → 生成订单 | **扣减库存**，生成地址快照 (防止后续地址修改影响) |
| **支付模拟**  | 模拟第三方支付回调          | 状态流转：待支付 → 待发货                         |
| **发货管理**  | 卖家填写物流单号            | 状态流转：待发货 → 已发货                         |
| **收货评价**  | 买家确认收货并评价          | 状态流转：已发货 → 已完成 → 已评价                |
| **售后/取消** | 取消订单                    | 仅未支付/未发货状态可取消                         |

### （5）互动与信用
| 功能         | 描述                                  |
| :----------- | :------------------------------------ |
| **商品收藏** | 收藏感兴趣的商品，方便后续查看        |
| **交易评价** | 交易完成后对商品/卖家进行打分 (1-5星) |
| **留言提问** | 在商品详情页公开发布提问              |

## 3. 技术架构

### 后端
- **核心框架**: Spring Boot 3.x
- **权限安全**: Spring Security + JWT
- **数据库**: MySQL 8.0
- **持久层**: MyBatis-Plus
- **对象存储**: 阿里云 OSS / MinIO
- **工具库**: Hutool, Lombok, Knife4j (接口文档)

### 前端
- **框架**: Vue 3 (Composition API) + TypeScript
- **UI组件**: Element Plus
- **状态管理**: Pinia
- **网络请求**: Axios (封装拦截器)
- **路由**: Vue Router 4

### 📚 系统核心数据字典与业务策略概述

#### 一、 核心状态数据字典 (Data Dictionary)

在前后端交互中，统一约定以下枚举值：

**1. 商品状态 (`product_status`)**

- `1`：**待审核**（用户刚发布，仅自己和管理员可见，展示在审核流水线）
- `2`：**已上架**（审核通过，全站可见，可被搜索和购买）
- `3`：**已下架/驳回**（用户主动下架，或管理员审核驳回/强制下架）
- `4`：**已售出**（交易完成，保留展示但不可购买）

**2. 商品新旧程度 (`condition_level`)**

- `1`：全新（吊牌未拆/未拆封）
- `2`：99新（仅拆封试用/几乎无使用痕迹）
- `3`：9成新（有轻微使用痕迹，功能完好）
- `4`：8成新及以下（有明显使用痕迹或微小瑕疵）

**3. 用户角色 (`role`)**

- `0`：**普通用户**（可买卖商品、评价、私聊）
- `1`：**系统管理员**（拥有商品审核、强制下架、用户封禁等最高权限）

------

#### 二、 基于信用分的奖惩与排序降权策略

**1. 基础设定**

- 初始信用分：`100` 分。
- 分数区间：最低 `0` 分，最高 `120` 分。

**2. 算分规则 (事件驱动)**

- **【加分项】**
  - 买家交易完成后给出**好评**：`+2` 分。
- **【扣分项】**
  - 商品因违规被管理员**驳回/强制下架**：`-5` 分。
  - 买家交易完成后给出**差评**：`-10` 分。
  - 买家或卖家在交易流程中**恶意取消订单**（若有该功能）：`-3` 分。

**3. 阈值惩罚机制**

- **信用优秀 (`>= 90分`)**：享受首页“猜你喜欢”兜底池的优先推荐权。
- **排序降权 (`< 80分`)**：触发流量限制。在任何分类搜索、全站列表查询中，无论其商品多新、价格多低，一律被强行置于搜索结果的末尾（沉底）。
- **限制发布 (`< 60分`)**：触发账号半封禁。禁止发布新的闲置商品，但保留购买和沟通权限。

------

#### 三、 基于用户行为权重的个性化推荐算法

本系统采用轻量级的“行为赋权分类召回模型”来替代复杂的协同过滤，兼顾性能与准确度。

**1. 行为权重得分计算** 用户在全站的每一个动作，都会为其在对应“商品分类 (`category_id`)”上累加偏好分：

- **浏览详情 (`View`)**：`+1` 分
- **加入收藏 (`Favorite`)**：`+3` 分
- **加入购物车 (`Cart`)**：`+5` 分
- **完成购买 (`Buy`)**：`+10` 分

**2. 推荐召回策略**

- **画像提取**：取该用户偏好总分最高的 Top 2 分类。
- **定向召回 (70%)**：从这 2 个分类中，提取最新发布的、状态为已上架的商品，占据推荐列表的 70% 份额。
- **探索发现 (30%)**：从用户非偏好的其他分类中，提取近期浏览量最高的热门商品，占据 30% 份额（打破信息茧房）。
- **列表打乱**：将召回的商品在内存中进行 Shuffle 打乱，返回给前端展示。

**3. 冷启动兜底策略** 对于没有任何行为记录的新注册用户，或未登录的游客：

- 提取全站浏览量最高、且**卖家信用分 `>= 90`** 的优质商品作为推荐列表。
