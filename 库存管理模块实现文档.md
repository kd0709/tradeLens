# 库存管理模块实现文档

## 概述
在闲置物品交易平台中实现了完整的库存管理功能，解决了原系统中缺少库存控制的问题，有效防止了商品超卖现象。

## 数据库设计回顾
根据系统概述文档，商品表 [product](file:///d:%5Czz-Final%5CtradeLens%5Cbackend%5Csrc%5Cmain%5Cjava%5Ccom%5Ccjh%5Cbackend%5Centity%5CProduct.java#L14-L89) 包含 `quantity INT DEFAULT 1 COMMENT '商品数量'` 字段，为库存管理提供了基础。

## 实现的功能

### 1. 库存检查机制
- 在创建订单时检查商品库存是否充足
- 如果库存不足则阻止订单创建并返回错误信息

### 2. 库存扣减机制
- 在支付宝异步回调确认支付成功后扣减库存
- 使用数据库层面的安全检查防止负库存 (`WHERE quantity >= #{quantity}`)

### 3. 库存回滚机制
- 在订单取消时（针对已支付订单）恢复库存
- 智能处理商品状态的回滚

### 4. 商品状态管理
- 订单确认收货后自动将商品状态更新为"已售"(4)
- 订单取消时在适当条件下将商品状态恢复为"上架"(2)

## 核心实现细节

### 数据库层面
在 `ProductMapper.xml` 中添加了关键的库存操作SQL：
- `decreaseProductQuantity`: 安全扣减库存，防止负库存
- `increaseProductQuantity`: 增加库存，用于订单取消
- `checkProductStock`: 检查可用库存，仅查询上架商品

在 `OrdersMapper.java` 中添加了 `countPaidOrdersByProductId` 方法用于智能状态管理。

### 业务逻辑层面

#### 订单创建阶段 ([OrdersServiceImpl.createOrder](file:///d:%5Czz-Final%5CtradeLens%5Cbackend%5Csrc%5Cmain%5Cjava%5Ccom%5Ccjh%5Cbackend%5Cservice%5Cimpl%5COrdersServiceImpl.java#L44-L104))
- 检查每个商品的库存是否满足订单数量要求
- 使用 `productMapper.checkProductStock()` 检查可用库存

#### 支付流程 ([OrdersServiceImpl.payOrder](file:///d:%5Czz-Final%5CtradeLens%5Cbackend%5Csrc%5Cmain%5Cjava%5Ccom%5Ccjh%5Cbackend%5Cservice%5Cimpl%5COrdersServiceImpl.java#L229-L258))
- 仅发起支付请求给支付宝
- 订单状态保持为待支付状态，等待支付宝回调

#### 支付宝回调处理阶段 ([AlipayController.handleNotify](file:///d:%5Czz-Final%5CtradeLens%5Cbackend%5Csrc%5Cmain%5Cjava%5Ccom%5Ccjh%5Cbackend%5Ccontroller%5CAlipayController.java#L32-L105))
- 收到支付宝支付成功通知后
- 扣减相关商品的库存
- 更新订单状态为已支付

#### 订单取消阶段 ([OrdersServiceImpl.cancelOrder](file:///d:%5Czz-Final%5CtradeLens%5Cbackend%5Csrc%5Cmain%5Cjava%5Ccom%5Ccjh%5Cbackend%5Cservice%5Cimpl%5COrdersServiceImpl.java#L140-L175))
- 对于已支付订单取消时恢复库存
- 智能判断是否需要将商品状态恢复为上架
- 使用 `countPaidOrdersByProductId` 方法避免误操作

#### 订单完成阶段 ([OrdersServiceImpl.confirmReceipt](file:///d:%5Czz-Final%5CtradeLens%5Cbackend%5Csrc%5Cmain%5Cjava%5Ccom%5Ccjh%5Cbackend%5Cservice%5Cimpl%5COrdersServiceImpl.java#L177-L205))
- 将订单相关商品状态更新为"已售"

## 设计特点

1. **安全性**：使用数据库约束防止负库存
2. **一致性**：通过事务保证数据一致性
3. **准确性**：在支付宝回调中进行真正的库存扣减，确保支付成功后再扣减库存
4. **智能性**：订单取消时智能判断是否需要恢复商品上架状态
5. **健壮性**：完善的异常处理和日志记录

## 关键改进

1. 将库存扣减从支付请求阶段移到支付宝异步回调阶段，确保只有真正支付成功的订单才会扣减库存
2. 添加了商品状态的智能管理，在订单生命周期中正确更新商品状态
3. 实现了安全的库存回滚机制，考虑了并发和多订单的情况

## 与原有逻辑的兼容性

- 所有修改均在不破坏原有功能的前提下进行
- 保留了原有的订单状态流转逻辑
- 仅在关键节点增加了库存检查和操作
- 所有新增功能都有异常处理，确保系统稳定性

该库存管理模块现已完全集成到现有系统中，能够有效防止超卖问题，提高系统的可靠性和用户体验，同时与原有逻辑完全兼容。
